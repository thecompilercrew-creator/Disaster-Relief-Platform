Thank you for considering contributing to the Disaster Relief Platform.

## How to Contribute

1. Fork the repository
2. Create a new branch
   ```bash
   git checkout -b feature/your-feature
   ```
3. Commit changes with clear messages
4. Push to your fork
5. Open a Pull Request

---

## Contribution Areas
- UI/UX improvements
- Backend optimization
- Map performance
- Accessibility
- AI enhancements
- Documentation

---

## Code Style
- Use meaningful variable names
- Avoid large monolithic functions
- Keep UI consistent with card-based design

---

## Reporting Issues
Include:
- Steps to reproduce
- Expected vs actual behavior
- Screenshots/logs if available

---

## Code of Conduct
- Be respectful
- Focus on constructive feedback
- Prioritize real-world impact

---

## Mission Reminder
This project exists to **help people during crises**. Contributions should align with that goal.
